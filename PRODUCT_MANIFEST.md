# Product Manifest — Interfacing

Interfacing is the ecosystem surface where users, operators, and administrators meet the system.

## Product scope
- public-facing surfaces;
- authenticated workspace surfaces;
- privileged operator/admin surfaces;
- shell, navigation, layout, widgets, screen contracts, runtime composition;
- Symfony-driven management UI for real component scenarios, not decorative pages only.

## Product promises
- one interface language across consumer and operator modes;
- fast delivery through reusable wrappers and zone contracts;
- governed styling without requiring manual design invention for each feature;
- Symfony-oriented runtime support for controllers, Twig, Bootstrap, Forms, Validator, and UX Live Components;
- future-ready upstream for PrimeReact facade enrichment and Ant Design workbench rigor.

## Product direction
- preserve the working server-driven runtime;
- enrich it with a governed UI upstream model;
- avoid duplicating front UI and admin UI into separate domains;
- keep the component demonstrable through meaningful screens, fixtures, CLI diagnostics, and browser-testable flows.
