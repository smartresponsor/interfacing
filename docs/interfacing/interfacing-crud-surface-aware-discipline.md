# Interfacing CRUD surface-aware discipline

The reusable CRUD renderer now reacts not only to route operation semantics but also to the `_crud_surface` dimension.

## Admin surface
- denser command bars
- explicit destructive affordances
- operator-oriented sidebar actions

## Public surface
- lighter command density
- safer destructive wording
- reduced sidebar action pressure
- same route semantics, softer rendering posture

This keeps Interfacing aligned with host CRUD routing while letting the center-body discipline present different UX intensity for admin versus public flows.
