# Interfacing surface audit

The surface audit is a shell-native transparency page for the Interfacing component.
It exists to make the interface layer explain itself instead of hiding transitional routes, sample links, or demo boundaries behind scattered pages.

## Route

```text
/interfacing/surface
```

## Scope

The page reports four groups:

- shell coverage: which operator pages use the canonical Interfacing base shell;
- CRUD route grammar: the exact URL patterns generated by the CRUD bridge;
- data boundary: where business rows and fixtures must come from;
- component coverage: which known Smart Responsor components are connected, canonical, or planned.

## Data boundary

Interfacing owns layout, navigation, rendering contracts, route grammar, empty states and operation affordances.
It must not own business fixtures for Cataloging, Ordering, Billing, Messaging, Taxating, Vendoring, Shipping or other components.

Sample show/edit/delete links may exist as navigation probes, but the sample identifier is not business data.
Real identifiers and records must come from the owning component.

## Status vocabulary

- `solid`: canonical surface or deliberate bridge;
- `watch`: accepted transitional surface that remains visible until replaced;
- `drift`: known wrong shape that should be corrected or removed.
